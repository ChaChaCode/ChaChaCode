# Как сделать себе такой же профиль

Это анимированный GitHub-профиль в стиле `neofetch`: одна SVG-картинка, в которую
GitHub Action раз в сутки подставляет свежую статистику (репозитории, коммиты, звёзды,
строки кода и т.д.). Ниже — как форкнуть его себе и настроить под свои данные.

---

## Шаг 1. Форк репозитория

1. Нажмите **Fork** вверху этого репозитория.
2. **Важно:** назовите форк **в точности как ваш GitHub-логин** (например, если вы
   `octocat`, репозиторий должен называться `octocat/octocat`). Только тогда GitHub
   покажет README на вашей странице профиля.

## Шаг 2. Создайте Personal Access Token (PAT)

Скрипту нужен токен, чтобы читать вашу статистику через GitHub API.

1. Откройте https://github.com/settings/tokens → **Generate new token (classic)**.
2. Отметьте права (scopes):
   - `repo` (весь блок) — чтобы считать приватные репозитории и строки кода
   - `read:user`
3. Сгенерируйте и **скопируйте** токен (он показывается один раз).

## Шаг 3. Добавьте секреты в репозиторий

В вашем форке: **Settings → Secrets and variables → Actions → New repository secret**.
Добавьте два секрета:

| Имя | Значение |
|-----|----------|
| `ACCESS_TOKEN` | ваш PAT из шага 2 |
| `USER_NAME`    | ваш GitHub-логин (например `octocat`) |

## Шаг 4. Впишите дату рождения

Она нужна для строки **Uptime** (ваш «аптайм» = возраст). Откройте
[`today.py`](today.py), найдите ближе к концу файла строку:

```python
age_data, age_time = perf_counter(daily_readme, datetime.datetime(2003, 7, 13))
```

Замените `2003, 7, 13` на свои **год, месяц, день**.

## Шаг 5. Отредактируйте текстовые поля

Числа (репозитории, коммиты, звёзды…) подставляются автоматически. А вот текстовые
строки — OS, стек, контакты — задаются вручную. Отредактируйте **оба** файла:
[`dark_mode.svg`](dark_mode.svg) и [`light_mode.svg`](light_mode.svg) — они должны
совпадать по тексту (различаются только цветами для тёмной/светлой темы).

Меняйте текст **справа от точек** в этих строках:

```
. OS ....................................... Windows 11, WSL2 (Ubuntu)
. Host ................................... Backend Developer · 5 years
. Kernel ............................... Node.js · NestJS · TypeScript
. IDE ................................................ VSCode, PyCharm
. Languages.Programming .......... TypeScript, JavaScript, Python, PHP
. Stack.Backend ........................ NestJS, Express, Node.js, PHP
. Stack.Data ...................... PostgreSQL, Prisma, TypeORM, Redis
. Stack.Realtime .............................. Socket.IO, SSE, WebRTC
. Hobbies.Software ................. Game Modding, Reverse Engineering
. DevOps.Prod ......................... Ubuntu VPS, nginx, PM2, Docker
. Email .......................................... your@email.com
. Telegram ................................................. @yourtag
. GitHub ....................................... github.com/yourname
. Location ............................................ Country (UTC+X)
```

### ASCII-арт слева (аватарка из символов)

Картинку слева можно сделать из любого своего изображения:

1. Загрузите фото на https://www.asciiart.eu/image-to-ascii и получите ASCII-версию.
2. **Важно: число строк арта должно совпадать с числом строк арта в SVG — сейчас это 25.**
   Если у вас получилось больше или меньше — подгоните высоту/масштаб на сайте, пока не
   станет ровно 25 строк. Иначе арт «съедет» относительно текста справа или вылезет за
   границы картинки.
3. Замените строки первого блока `<text>` (левый столбец, `class="ascii"`) на свои. Каждая
   строка — это отдельный `<tspan>` со своей координатой `y` (30, 50, 70 … 510, шаг 20).
   Просто вставьте текст ASCII-арта внутрь `<tspan>…</tspan>`, не трогая сами теги и `y`.

Сделайте это в **обоих** файлах (`dark_mode.svg` и `light_mode.svg`).

## Шаг 6. Обновите README

В [`README.md`](README.md) замените `ChaChaCode` на свой логин в трёх местах — иначе
будет показываться чужая картинка:

```html
<a href="https://github.com/ВАШ_ЛОГИН/ВАШ_ЛОГИН">
  <picture>
    <source media="(prefers-color-scheme: dark)" srcset="https://raw.githubusercontent.com/ВАШ_ЛОГИН/ВАШ_ЛОГИН/main/dark_mode.svg">
    <img alt="GitHub Profile README" src="https://raw.githubusercontent.com/ВАШ_ЛОГИН/ВАШ_ЛОГИН/main/light_mode.svg">
  </picture>
</a>
```

## Шаг 7. Запустите

Закоммитьте изменения — Action запустится сам (он также работает по расписанию раз в
сутки и по кнопке **Actions → README build → Run workflow**). Через минуту-две картинка
на вашем профиле обновится.

> Картинка на profile-странице кэшируется GitHub (camo) на несколько минут — если сразу
> не видно изменений, подождите и обновите страницу через **Ctrl+Shift+R**.

## Как устроено выравнивание (если хотите менять раскладку)

Текст — моноширинный, а колонки выравниваются **точками-заполнителями**.

- **Числа статистики выравниваются сами.** Функция `justify_format(root, id, value, length)`
  в [`today.py`](today.py) ставит `length − len(value)` точек. Поэтому если, например,
  звёзд станет `223` вместо `1`, скрипт сам уберёт 2 точки — правый край не сдвинется.
  Целевые ширины колонок задаются аргументом `length` в этих вызовах (ищите
  `justify_format(...)` в `today.py`).
- **Текстовые строки (OS, стек, контакты) — точки статичные.** Если поменяете значение на
  более длинное/короткое, поправьте число точек вручную, чтобы правый край остался ровным.
- **Правый край.** Все строки правого блока имеют одинаковую ширину (≈70 моноширинных
  символов) — так они доходят до конца линий-заголовков `───`. Символ `─` уже обычного,
  поэтому по длине линии судить о ширине нельзя — ориентируйтесь на текстовые строки.
- **Разделители `|` в блоке GitHub Stats** должны стоять в одной колонке во всех трёх
  строках. Если двигаете точки — двигайте одинаково во всех трёх, иначе `|` разъедутся.
